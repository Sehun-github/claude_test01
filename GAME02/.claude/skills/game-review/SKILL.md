## 검토 가능한 게임 파일
!`ls *.html 2>/dev/null`

검토 대상: `$ARGUMENTS` (비어있으면 위 목록에서 어떤 파일을 검토할지 사용자에게 확인)

## 검토 항목
1. **버그** — 게임 로직 오류, 예외 처리 누락, 무한 루프 가능성
2. **UX** — 조작감, 피드백(애니메이션), 게임오버/클리어 처리
3. **모바일 대응** — 터치 이벤트, 뷰포트, 버튼 크기
4. **성능** — requestAnimationFrame 사용 여부, 메모리 누수, 과도한 DOM 조작
5. **스타일 일관성** — 프로젝트 디자인 시스템 준수 여부
6. **게임 밸런스** — 난이도 곡선, 재미 요소

## 절차
1. 대상 파일 전체를 Read로 읽고 위 항목별로 분석
2. 심각도(높음/중간/낮음)와 함께 문제점 목록 출력
3. 수정 제안을 구체적으로 제시
