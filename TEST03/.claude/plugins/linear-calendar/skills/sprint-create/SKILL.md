# /sprint-create

스프린트 목표를 입력받아 관련 이슈들을 한번에 생성하고 캘린더에 등록합니다.

## 사용법

```
/sprint-create
```

실행 후 사용자에게 다음을 질문하세요:
1. 스프린트 이름 (예: "로그인 기능 구현")
2. 기간 (시작일 ~ 종료일)
3. 프로젝트 선택 (mcp__linear__list_projects 로 목록 조회 후 제시)
4. 주요 작업 목록 (자유 입력, 없으면 AI가 제안)

## 실행 순서

1. `mcp__linear__list_projects` 로 프로젝트 목록 조회
2. 사용자 입력을 바탕으로 이슈 목록 생성 (없으면 스프린트 목표에서 추론)
3. 각 이슈를 `mcp__linear__create_issue` 로 생성
   - status: todo
   - 마감일: 스프린트 종료일 또는 단계별 배분
4. 마감일이 있는 이슈는 캘린더에 자동 등록됨 (Linear-Calendar 연동)
5. 스프린트 킥오프 일정을 `mcp__calendar__create_event` 로 추가

## 출력 형식

```
🚀 스프린트 생성 완료: [스프린트 이름]
기간: MM/DD ~ MM/DD

생성된 이슈 (N개):
✅ [FE-4] 이슈 제목 (마감: MM/DD)
✅ [BE-3] 이슈 제목 (마감: MM/DD)
...

📅 캘린더 등록:
✅ 스프린트 킥오프 (MM/DD)
✅ 각 이슈 마감일 자동 등록됨
```
